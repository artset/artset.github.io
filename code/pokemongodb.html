PokemonGo Filter

Autumn 2019

React, Python

Context

PokemonGo is a VR mobile game where users can walk around the real world and encounter Pokemon. Pokemon can be found in a variety of ways, from catching to hatching eggs, to raid events. Additionally, the elusive Ditto can only be found disguised as other Pokemon. Given that these events cycle every few weeks, active players of the game must scour the internet (cough, Reddit) to find out if their favorite Pokemon is available in a raid, or what Pokemon are available in the egg pool. In other words, there is no place where all the information is in one place.

To solve this issue, my friend Julie and I developed a web app where users can filter Pokemon by type, as well as specific categories relevant to events. We built this web app specifically for users of the game, who can easily access the most commonly requested information, such as type and egg pool.

View the demo!

Process

For this project, I scraped Pokemon names, dex names, and images from TheSilphRoad and types from PokemonDB. Julie acquired the data for raids and ditto from LeekDuck. After, I was responsible for implementing the card design, accordion feature, and the favorite feature. We collaborated on usability principles, as well as core visual components of the page, such as typeface and color.

Usability Principles

We used different UI components, such as drop down vs a minimalized version of a checkbox, to convey different purposes. In the case of “ascending/descending”, only one choice is possible, which led us to choose to use a dropdown box for this feature, since the dropdown only shows the option selected.

For filtering through types (electric, water, etc), we emulated the logic of how multiple checkboxes can be selected with a more minimal UI. In our implementation, the background becomes grey when these filters are selected, giving visual feedback of the user’s action. Multiple buttons are allowed to be selected given the user can filter many types.

Additionally, we incorporated an accordion style navigation so the user can easily sift through the filters they want to use. The accordion also serves to declutter the window, so the user can decide what information is important to them at the moment.

Moving forwards

Currently, the web app is manually updated via json files, causing it to easily out of date. It would be great if there was a way to update the database easily, or if the information was crowd sourced. Additionally, the load time is fairly slow given that there are 600+ Pokemon that need to be rendered. For the future, I would like to add lazy load to solve this issue. Overall, I really loved working on this project and hope to dabble more with React!
